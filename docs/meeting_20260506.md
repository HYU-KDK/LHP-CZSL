## 2. 핵심 진단 결과 (5/4)

### 2-1. Prototype collapse — baseline이 K를 안 씀

| ckpt | attr eff_K | attr off-cos |
|---|---:|---:|
| ClusPro baseline mit | 1.08 / 5 | 0.994 |
| ClusPro baseline UT-Zap (pre-fix) | 1.06 ~ 1.37 | 0.97 ~ 0.999 |
| LHP v3_text UT-Zap (text-ensemble) | **3.88 ~ 3.89** | 0.015 ~ 0.025 |

→ baseline은 5개 prototype이 사실상 동일 vector(eff_K≈1)였음.

### 2-2. Gumbel-Softmax fix → baseline 회복

| ckpt | attr eff_K | attr off-cos |
|---|---:|---:|
| baseline v2 UT-Zap (**pre**-fix) | 1.06 ~ 1.37 | 0.967 ~ 0.999 |
| baseline v2 UT-Zap (**post**-fix) | **2.21 ~ 2.66** | 0.728 ~ 0.833 |

→ ClusPro 공식 코드(hard one-hot 강제)를 매칭하니까 K 사용 회복.
즉 **이전 baseline 수치는 K=1짜리였음**.

### 2-3. K-validation: LLM K vs visual K

UT-Zap (n=28 primitives): **ρ = −0.45**
LLM이 더 많이 쪼갠 primitive일수록 visual-cluster K는 더 작음 (anti-correlated).
→ LLM K는 visual diversity의 proxy로 못 씀.

## 3. UT-Zap 학습 결과 (post-gumbelfix, 3-seed 확정)

mean HM/AUC ± std over 3 seeds, val_best.pt @ val_pairs HM peak:

| variant | mean HM | mean AUC | eff_K (attr / obj, post-fix) |
|---|---:|---:|:---:|
| baseline v2 | **0.6694 ± 0.001** | **0.5201 ± 0.004** | 2.44 / 1.62 |
| LHP v1_init_only | **0.6646 ± 0.013** | **0.5113 ± 0.020** | **3.89 / 3.74** |
| LHP v3_text | **0.6735 ± 0.005** | **0.5272 ± 0.004** | **3.89 / 3.74** |

- **v1_init_only ≈ baseline (HM −0.005, AUC −0.009)** — visual-side variable-K prototype은 효과 미미
- **v3_text > baseline (HM +0.004, AUC +0.007)** — text-side ensemble이 작지만 일관된 gain
- **eff_K**: baseline post-fix가 1.06–1.37 (pre-fix) → 2.44 (post-fix)로 회복. LHP variants는 3.89/3.74로 prototype 더 잘 분리하지만 **성능은 tie** → "prototype 분리도가 성능 lever 아님" 결론 단단해짐.

### 3.1 Open-world (Phase 1 결과, 3-seed × {mask on, mask off}, test_pairs)

| variant | closed-world | open-world (mask OFF) | open-world (mask ON) |
|---|---|---|---|
| baseline v2 | 0.5374 / 0.4131 | **0.4137 / 0.2804** | 0.4010 / 0.2653 |

**Mask 기여도 — 3-seed Δ (mask on − mask off, test_pairs):**

| variant | ΔHM ± std | **ΔAUC ± std** | 신뢰도 |
|---|---|---|---|
| baseline v2 | −0.013 ± 0.002 | **−0.015 ± 0.002** | ≈ 7σ |

## 8. 선행 연구 비교 + 차별화 axis 분석 (5/5 추가)

### 8-1. 거의 똑같은 선행 논문이 있음

**FLM** — *"Feasibility with Language Models for Open-World Compositional
Zero-Shot Learning"* — Kim, Alaniz, Schmid, Akata (ECCV-W OOD-CV 2024,
arXiv 2505.11181). LLM에게 (state, object) 페어 feasibility 물어보고
post-hoc filter로 사용. Vicuna/ChatGPT, MIT-States/UT-Zappos/C-GQA에서
SOTA 갱신.

**우리와 차이**:

| | FLM | 우리 |
|---|---|---|
| LLM | Vicuna / ChatGPT | Gemini 2.5-flash-lite |
| 출력 형태 | yes/no logit (확률) | 0~10 정수 점수 |
| 프롬프트 | in-context 예시 다수 | scoring guide만 |
| 통합 방식 | post-hoc threshold | post-hoc threshold (`test.py:508`, 동일) |
| 모델 변경 | 없음 (baseline 그대로) | LHP-CZSL sub-meaning prototype |

→ **단순 LLM-feasibility 자체는 contribution 약함**. 차별화 axis 필요.

### 8-2. 그 이전 계보 (LLM 아닌 feasibility)

- **Mancini et al. 2021 (CVPR)** — OW-CZSL setup 처음 제안. GloVe 유사도로
  feasibility 계산. `test.py:508 threshold_with_feasibility`가 이 논문 코드.
- **Mancini et al. 2022 (TPAMI)** — Co-CGE. feasibility를 cosine margin +
  graph weight로 학습 단계까지 주입.
- **Karthik et al. 2022 (CVPR)** — KG-SP. ConceptNet 등 지식그래프 기반.

### 8-3. 차별화 후보 axis (강도 순)

#### ★★★★★ — 논문 한 편 분량

**(A) Prototype × Feasibility 직교성 분석**
- FLM은 baseline 위 post-hoc filter만. 모델 안 건드림.
- 우리는 LHP-CZSL로 모델 자체를 바꿈.
- 4-way ablation: {baseline, LHP v3_text} × {closed, +feasibility(open)}.
  - prototype gain이 open으로 그대로 transfer되고 + feasibility α도 더해지면
    → additive contribution. 깨끗한 main paper.
  - prototype gain이 feasibility 적용 후 사라지면
    → "LLM feasibility가 prototype 메커니즘을 subsume한다" 라는 중요한
    negative finding (workshop publishable).
- **현재 셋업으로 즉시 측정 가능** (v3_text 학습 끝나는 대로).

**(B) Image-conditional feasibility (VLM 기반)**
- FLM 포함 모든 기존 연구의 한계: feasibility가 image-agnostic prior.
- VLM(Gemini Pro Vision, GPT-4V)에 image + 후보 페어를 함께 보여주고
  per-image feasibility 산출. → tensor [N_test_images × N_pairs].
- test set만큼만 호출하면 되니 비용 합리적. 새 axis라 FLM 한계 직접 공격.

#### ★★★★ — 안전한 차별점

**(C) Training-time LLM signal (post-hoc 탈피)**
- FLM/KG-SP는 모두 inference-time filter. Co-CGE만 GloVe로 학습 단계 손댐
  (LLM 아님).
- LLM feasibility를 contrastive sampling weight / soft target / curriculum로
  학습에 주입. Architecture 작업 1~2주.

**(D) LLM 점수 형식·calibration 비교 연구**
- yes/no logit (FLM) vs 0~10 정수 (우리) vs pairwise comparison vs
  Likert + CoT.
- 각 형식이 실제 unseen pair 빈도와 어떻게 correlate되는지 spearman ρ.
- 단독 contribution은 약함. (A)나 (B) 안의 ablation 챕터로 강화.

#### ★★★ — 보조

**(E) LLM 멀티시그널 활용 (feasibility를 넘어)**
- per-pair visual schema, fine-grained sub-meaning, negative cue 자동 생성,
  실패 페어 공통 패턴 분석. "LLM as multi-tool for OW-CZSL" framing.

**(F) 새 LLM·새 데이터셋 데이터포인트**
- Gemini-2.5 / Claude-Sonnet-4.5 / GPT-4o 비교.
- C-GQA / MIT 재검증, 새 OW 벤치마크.

#### ★★ — 현실 인식

**(G) Negative result 분석 논문**
- (A) 결과가 "feasibility 적용 후 prototype gain 사라짐"이면 그 자체가
  community reality check 가치. 여러 prototype-method에 동일 filter 적용해서
  방법 간 차이가 얼마나 남는지 측정.

### Phase 3 — 중기 (2~4주, **Phase 1 negative 확정**)

Phase 1 결과 확정으로 **negative 분기 활성** (α' = −0.015 AUC, 통계 유의).

#### 우선순위 1 — axis B 검증 후 가지 선택:

axis B (image-conditional VLM)을 1주 안에 simplest setup으로 검증하고, 효과 크기에 따라 분기:
- B가 baseline mask off 대비 **+ε > 시드 std** → axis B 자체가 main contribution이 됨. axis A의 prototype 변형은 ablation chapter로 흡수.
- B도 ≈ 0 또는 음수 → axis G (negative result paper) 본격화.

#### 우선순위 2 — axis G (Reality-check) 본격화 (B 결과 본 후):

**[G] Reality-check 논문 framing**
- 5~10개 prototype-based CZSL method (CSP, Troika, ClusPro, LHP variants 등)에 동일 LLM feasibility filter 적용
- "feasibility 후 method 간 차이가 얼마나 남는가" 측정
- 우리 Phase 1 데이터(3 모델, 3 seed, mask on/off)는 이미 G의 일부 → 5~7개 더 추가하면 핵심 표 완성
- Workshop 또는 ablation-heavy 논문으로 publishable

#### 보류 (Phase 1 negative라 우선순위 낮음):

**[C] Training-time LLM signal** ★★★★ — postponed
- prototype 메커니즘 자체가 lever 아님이 확인된 마당에, training-time LLM signal로도 큰 격차 기대 어려움.
- axis B 결과가 positive로 나오면 그때 [B + C 결합]으로 재기.

### Phase 4 — 장기 (1~2개월)

**[E] LLM 멀티시그널 활용** ★★★
- per-pair visual schema → CLIP text encoder enrichment

- "LLM as multi-tool" framing으로 main story extension

**[F] 데이터·LLM 스케일링** ★★★
